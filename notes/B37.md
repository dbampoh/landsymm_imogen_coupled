## 0. Headline finding (1-paragraph TL;DR for future maintainers)

The IMOGEN engine's productive-year ceiling under single-process `imogencfx` mode + F-10 case-α deadlock is **NOT** a stochastic empirical mystery — it is a **deterministic closed-form function of the bootstrap YEAR1**, governed by three hardcoded year sentinels (`1900`, `2100`, `1871`) embedded in `lpjguess/modules/climatemodel.cpp::updateImogenControlData()` at lines 1185-1197. The ceiling formula `productive_years = 2 × (⌈(1900 - YEAR1 + 1) / 2⌉ + 1)` matches both prior empirical observations (Run B 1871-start → 32 years; Run C 1900-start → 4 years) exactly. **Three candidate paths from the original B37 row were investigated** (path i = configurable ceiling; path ii = `--coupling-mode loose` bypass; path iii = `skip_inprocess_engine_run` two-step) — **PLUS a NEW path (iv) launcher-side `done`-marker sidecar emerged from the source-reading**. Path (iv) was empirically confirmed in DR1: **the IMOGEN engine produced ALL 201 paper-publication-target years (1900-2100) plus 1 over-shoot placeholder (2101; benign cosmetic; ignorable), in ~12.6 min wall on the smoke 4-cell config, with no source-code change required**. The v1.0 paper-publication production-IMOGEN runs (full 1900-2100 × 5 SSP-RCP scenarios) are **NOT blocked on F-12 architectural fix**. Recommended productisation of path (iv) via a small `scripts/run_coupled.sh` enhancement is filed as **NEW B44** (~20-30 LOC bash; MEDIUM priority); recommended cleanup of the 3 hardcoded year sentinels in `updateImogenControlData()` is filed as **NEW B45** (~5-15 LOC source-edit; LOW priority; cosmetic + TRUNK-RELEVANT for the canonical engine; can wait for v1.1+ alongside F-12 work).

## 2. Source-reading verdict (Phase 1)

### 2.1 The state-machine that controls the ceiling

The C++ in-process IMOGEN engine at `lpjguess/modules/climatemodel.cpp::RUN_IMOGEN_ENGINE` (line 153) is a **3-level state machine**:

| Level | Code site | Purpose |
|---|---|---|
| Outer `while(keepRunning)` | line 336 | one engine-call worth of years per iteration; iterates until `KEEPRUNNING=false` |
| Polling loop `while(!runnow)` | line 347 | waits for LPJG handshake files (`done` + `imogen_lpjg.txt` + `imogen_lpjg_flux.txt` + optionally `imogen_lpjg_ch4_n2o_flux.txt`). **Bypassed when LOCAL `firstCall=TRUE`** via the special-case at lines 358-361 (the `firstCall && !doneExist → doneExist=true` bypass installed by step-7 bug-C2 fix) |
| Inner `for(iyear=year1; iyear<=iyend; ++iyear)` | line 474 | processes years from YEAR1 to IYEND inclusive; calls `updateImogenControlData()` at the end of each inner iteration |

At the end of each inner-iyear iteration, `updateImogenControlData()` (line 1181) executes:

```1181:1199:lpjguess/modules/climatemodel.cpp
void updateImogenControlData() {
    IMOGENConfig::YEAR1++;
    IMOGENConfig::IYEND++;

    if (IMOGENConfig::YEAR1 == 1901) {
        IMOGENConfig::SPINUP=false;
    }

    if (IMOGENConfig::YEAR1 == 2100) {
        IMOGENConfig::KEEPRUNNING = false;
    }

    if (IMOGENConfig::IYEND > 1900) {
        IMOGENConfig::FIRSTCALL = false;
    }

    IMOGENConfig::YEAR1_LPJG = 1871;

}
```

**Three hardcoded year sentinels** (`1901`, `1900`, `2100` + the `1871` reset) embedded in pure C++ logic — none parametrised against the `.ins`. The brittle ones for the ceiling discussion are:

- **`if (IMOGENConfig::IYEND > 1900) FIRSTCALL=false`** at line 1193 — this is the root cause of the productive-year ceiling.
- **`if (IMOGENConfig::YEAR1 == 2100) KEEPRUNNING=false`** at line 1189 — this is the over-shoot-by-one mechanism that creates the 2101 dir.
- **`IMOGENConfig::YEAR1_LPJG = 1871`** at line 1197 — this hardcoded reset is what causes the engine's over-shoot iteration to try opening `./IMOGEN/output/1871/T_anom.dat` (an artifact of an old design that's no longer relevant for 1900-start configs).

### 2.2 The mechanism explaining the ceiling

In single-process tight-coupling, **LPJG main loop never runs** (because `imogencfx::init()` calls `RUN_IMOGEN_ENGINE` inline at line 549 and never returns until the engine exits). So no LPJG ever writes a real `done` marker. The engine relies ENTIRELY on the FIRSTCALL bypass at lines 358-361 to make the polling loop pass without an LPJG `done`.

The LOCAL `firstCall` variable in the polling loop is captured at line 410 of the **PREVIOUS** outer iteration — line 410 fires AFTER polling succeeds and BEFORE the for-loop runs:

```cpp
year1 = IMOGENConfig::YEAR1;
iyend = IMOGENConfig::IYEND;
spinup = IMOGENConfig::SPINUP;
year1Lpjg = IMOGENConfig::YEAR1_LPJG;
keepRunning = IMOGENConfig::KEEPRUNNING;
firstCall = IMOGENConfig::FIRSTCALL;
```

So if updateImogenControlData() flips IMOGENConfig::FIRSTCALL=false during iter K's for-loop (after IYEND first exceeds 1900), the LOCAL firstCall in iter K+1's polling loop is STILL TRUE (because it was captured at line 410 in iter K BEFORE the flip). Iter K+1's polling bypass still works. Then line 410 in iter K+1 captures FIRSTCALL=FALSE; iter K+2's polling loop has LOCAL firstCall=FALSE; bypass FAILS; engine deadlocks forever.

### 2.3 Closed-form formula

For YEAR1 = Y with IYEND = Y+1 (per the launcher's bootstrap at `scripts/run_coupled.sh:363-364`):

- **K_flip** = ⌈(1900 - Y + 1) / 2⌉ — first outer iter whose first inner `updateImogen()` call advances IYEND past 1900 (sets `IMOGENConfig::FIRSTCALL=false`)
- **Productive outer iters** = K_flip + 1 — lag-1 effect of the LOCAL firstCall capture happening at line 410 BEFORE the iter's for-loop
- **Productive years** = 2 × (K_flip + 1) — since launcher bootstraps IYEND = YEAR1 + 1

### 2.4 Empirical match table

| YEAR1 | K_flip | Productive iters | Productive years | Empirical evidence | Match? |
|---|---|---|---|---|---|
| 1850 | 26 | 27 | 54 | (predicted; not observed) | — |
| **1871** | **15** | **16** | **32** | **Run B = 32 years (1871-1902)** | ✅ |

| **1900** | **1** | **2** | **4** | **Run C = 4 years (1900-1903)** ✅ + **DR2 control 2026-05-19 = 4 years** | ✅✅ |
| 1901 | 0 | 1 | 2 | (predicted; not observed) | — |
| 2000 | 0 | 1 | 2 | (predicted; not observed) | — |

**Conclusion**: the productive-year ceiling is a deterministic function of YEAR1; the 8× discrepancy between Run B and Run C is fully explained by the `IYEND > 1900` hardcoded threshold at `climatemodel.cpp:1193`.

### 2.5 Path-by-path source-reading verdict

#### Path (i) Configurable productive-year ceiling — **VIABLE**

**Fix variants**:
- **(α)** Remove or relax the `IYEND > 1900` check at line 1193 entirely → FIRSTCALL stays TRUE forever → engine runs unbounded until KEEPRUNNING flips false (which itself is hardcoded at YEAR1==2100 in line 1189; needs relaxing too).
- **(β)** Make all 3 hardcoded year sentinels (`1901`, `2100`, `1871`) configurable per `.ins` via new parameters (e.g., `SPINUP_END_YEAR 1901`, `RUN_END_YEAR 2100`, `YEAR1_LPJG_RESET 1871` or remove the reset entirely). ~10-20 LOC source-edit + ~10 LOC `.ins`-side defaults + cross-reference table in `docs/scientific_framework.md`.
- **(γ)** Add a single new parameter `BYPASS_LPJG_HANDSHAKE true` that makes the polling loop ALWAYS bypass when done doesn't exist. ~5 LOC.

**Effort**: 3-30 LOC source-edit; TRUNK-RELEVANT for the C++ port; if applied to the Fortran twin at `imogen/code/imogen_lpjg.f` it becomes TRUNK-RELEVANT for both forks.

**Operational reality**: this fix essentially defeats the FIRSTCALL design intent. The FIRSTCALL bypass was meant as a one-shot "this is the very first call, no handshake yet" mechanism; removing the 1900 cutoff makes it permanent. That's semantically equivalent to "the engine runs without LPJG handshake forever", which is operationally what we want for v1.0 paper-publication prescribed-mode runs.

**Verdict**: ✅ VIABLE — but requires source modification + careful semantic re-documentation. Filed as **NEW B45 candidate** for v1.1+ source-edit cleanup.

#### Path (ii) `--coupling-mode loose` bypass — **NOT VIABLE for `-input imogencfx`**

**Source evidence**:
- `IMOGENConfig::coupling_mode` is declared at `lpjguess/modules/imogencfx.cpp:340` + `lpjguess/framework/parameters.h:504` + `lpjguess/framework/parameters.cpp:279`.
- Consumed at `lpjguess/modules/imogenoutput.cpp` lines 163 (init banner), 269-274 (`flush_year` early-return when mode=='loose'), 473-478 (`flush_year_globally_synchronized` early-return when mode=='loose').
- **NOT consumed inside `RUN_IMOGEN_ENGINE`** in `climatemodel.cpp` (verified via grep; only consumer is `imogenoutput.cpp`).

**Implication**: `--coupling-mode loose` only affects the **LPJG-side handshake writer** (which is in any case unreachable in single-process tight because LPJG main loop never runs). It does NOT affect the engine's polling loop. The engine still deadlocks at the same FIRSTCALL boundary.

**Note**: `--coupling-mode loose` IS viable in the legacy framework's sense — with `-input imogen` (different input module that reads pre-baked climate from disk; legacy compatibility per `docs/scientific_framework.md` §1). But that's NOT Track 2's `-input imogencfx` requirement.

**Verdict**: ❌ NOT VIABLE for the v1.0 paper-publication target.

#### Path (iii) `skip_inprocess_engine_run` two-step workflow — **PARTIALLY VIABLE; requires upstream solution for pre-stage step**

**Source evidence**:
- Parameter declared at `lpjguess/modules/imogencfx.cpp:365`; used at line 548-549 to gate the `RUN_IMOGEN_ENGINE()` call.
- When TRUE, the engine call is skipped; user must pre-stage climate at `<DIR_COMMON>/IMOGEN/output/<YYYY>/`.
- Per the parameter's docstring + the comment block at lines 355-364: "Used for `-input imogencfx` cross-validation per F-12 sub-step 7.3.2".

**The pre-stage problem**: how does one produce 201 year-dirs of climate WITHOUT running LPJG? Options:
- **(iiia)** Use the standalone Fortran engine binary at `imogen/code/imogen_lpjg` — confirmed exists + builds (sha1 verified). But: the Fortran source at `imogen/code/imogen_lpjg.f` has the **same polling-loop architecture** (lines 391-457: `DO WHILE (KEEPRUNNING)` outer + `DO WHILE (RUNNOW.EQV..FALSE.)` polling + `IF (RUNNOW_EXIST.AND.RUNFLUX_EXIST.AND.DONE_EXIST) ... CALL SETTIN_LPJG ... RUNNOW=.TRUE.`). So the standalone Fortran engine deadlocks without LPJG too (subject to a Fortran-specific FIRSTCALL handling — Fortran reads FIRSTCALL from `imogen_lpjg.txt` each iter via SETTIN_LPJG which might allow per-iter override, but not verified empirically here).
- **(iiib)** Apply path (i) fix to C++ engine, then split: (1) run guess with `skip_inprocess_engine_run=false` to produce 201 year-dirs (engine runs to completion via path-i fix); (2) run guess with `skip_inprocess_engine_run=true` to consume pre-staged climate. **But**: if path (i) already makes the engine complete in-process, there's no need to split — the single-pass run produces the year-dirs AND then LPJG can consume them within the same process.
- **(iiic)** Use the NEW path (iv) sidecar approach to pre-stage, then `skip_inprocess_engine_run=true` for LPJG consumption.

**Verdict**: ⚠️ PARTIALLY VIABLE — the parameter itself works as designed, but the upstream pre-stage step requires solving the engine-without-LPJG problem first (which is what path (i) or path (iv) does).

#### Path (iv) Launcher-side `done`-marker sidecar — **NEW; VIABLE; empirically confirmed**

**Source-reading discovery**: at `climatemodel.cpp:347-401`, the polling loop sleeps 3 seconds per cycle and rechecks for `done` file existence. After polling succeeds and handshake files are read, the engine **deletes `done`** at line 578 (`filesystem_dkb::remove(dirCommon + "/LPJG_main/IMOGEN/done")`). If `done` is **periodically re-created** by an external process (e.g., a launcher-side sidecar bash that touches the file every 1 second), the engine sees `done` on subsequent poll cycles and progresses through ALL years without needing the FIRSTCALL bypass.

**Effort**: ZERO source change; ~5-10 LOC bash sidecar.

**Verdict**: ✅✅ VIABLE — and empirically confirmed in DR1 (see §3.2 below).

**Trigger**: B37 Phase 2 empirical confirmation that a simple bash sidecar (`while true; do touch <HSHAKE>/done; sleep 1; done`) running alongside `run_coupled.sh` enables the IMOGEN engine to produce all 201 paper-publication-target years (1900-2100) without F-12 architectural fix.
